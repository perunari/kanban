# Security Assumptions

## このアプリケーションのセキュリティ前提条件

### 対象環境

このカンバンアプリは **社内ネットワーク限定** での使用を前提としています。

### ネットワーク境界保護（必須）

以下のいずれかの手段でネットワーク境界を保護してください:

- **VPN**: アクセス可能なユーザーを VPN 接続済みの社内ユーザーに限定する
- **IP 制限**: ファイアウォール・リバースプロキシ（nginx, AWS ALB 等）で許可 IP を制限する
- **プライベートネットワーク**: インターネットから到達不可能なプライベートサブネットに配置する

**上記の保護なしにインターネットへ公開しないでください。**

### VITE_API_KEY のリスク

`VITE_API_KEY` は Vite のビルド時に JavaScript バンドルへ **静的に埋め込まれます**。

- ブラウザの DevTools（Sources タブ）からバンドルを閲覧すれば誰でも API キーを取得できます
- 取得したキーを使ってバックエンド API へ直接アクセスが可能になります

このリスクは **ネットワーク境界での保護**（VPN・IP 制限）によって緩和します。  
インターネットに公開する場合は BFF (Backend For Frontend) パターンへの変更が必要です。

### 認証・認可

| 項目 | 実装状況 |
|------|---------|
| API キー認証 | 全 `/api/*` エンドポイントで必須 |
| `/health` エンドポイント | 認証不要（死活監視用） |
| API_KEY 未設定時 | バックエンドは起動を拒否 |

### その他のセキュリティ対策

- SQL インジェクション: プレースホルダー (`?`) を全クエリで使用
- XSS: React/Ant Design による自動エスケープ（`dangerouslySetInnerHTML` 不使用）
- CORS: `ALLOWED_ORIGIN` 環境変数で制御
- 入力バリデーション: 型・長さ・ホワイトリストチェックを全エンドポイントで実施
- エラーレスポンス: スタックトレース等の内部情報を含まない
